# GitHub Auto Commit Tool

Программа для автоматизированного заполнения графика contributions на GitHub через создание коммитов с разными датами.

## Возможности

- ✅ Автоматическое заполнение графика contributions за указанный период
- ✅ Коммиты с указанием даты (включая прошлые даты)
- ✅ Настраиваемые паттерны активности (случайный, будние дни, постоянный)
- ✅ Автоматическая отправка изменений на GitHub
- ✅ Гибкая конфигурация через JSON файл

## Установка

1. Клонируйте или скачайте репозиторий

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```

4. Получите GitHub Personal Access Token (PAT):
   
   **Важно:** Нужен именно Personal Access Token (классический), не Fine-grained token!
   
   Шаги:
   - Перейдите на https://github.com/settings/tokens
   - Нажмите "Generate new token" → "Generate new token (classic)"
   - Дайте токену имя (например, "Auto Commit Tool")
   - Выберите срок действия (expiration)
   - **Обязательно отметьте права (scopes):**
     - ✅ `repo` (полный доступ к репозиториям) - это включает:
       - `repo:status`
       - `repo_deployment`
       - `public_repo`
       - `repo:invite`
       - `security_events`
   - Нажмите "Generate token"
   - **ВАЖНО:** Скопируйте токен сразу! Он показывается только один раз
   - Вставьте токен в файл `.env`:
   ```
   GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
   
   **Примечание:** Если вы используете Fine-grained token (новый тип), убедитесь что у него есть права `Contents: Write` и `Metadata: Read`

5. Настройте `config.json` под свои нужды

## Использование

### Базовое использование

1. Убедитесь, что вы находитесь в git репозитории или укажите путь в `config.json`

2. Запустите программу:
```bash
python main.py
```

3. Программа создаст коммиты за указанный период и отправит их на GitHub

### Настройка конфигурации

Отредактируйте `config.json`:

```json
{
  "repository_path": ".",  // Путь к репозиторию (точка = текущая директория)
  "fill_contributions": {
    "enabled": true,                    // Включить заполнение графика
    "start_date": "2024-01-01",         // Начальная дата
    "end_date": "2024-12-31",           // Конечная дата
    "min_commits_per_day": 1,           // Минимум коммитов в день
    "max_commits_per_day": 10,          // Максимум коммитов в день
    "activity_pattern": "random",       // Паттерн: "random", "weekday", "consistent"
    "weekend_activity": true            // Активность в выходные дни
  },
  "auto_push": true                     // Автоматически отправлять на GitHub
}
```

### Паттерны активности

- `random` - случайное количество коммитов каждый день
- `weekday` - больше активности в будние дни
- `consistent` - более равномерное распределение

## Важные замечания

⚠️ **Внимание**: Использование автоматизированных коммитов может нарушать Terms of Service GitHub. Используйте на свой риск.

- Коммиты создаются в директории `activity/` репозитория
- Для отображения contributions на GitHub необходимо сделать push
- GitHub обновляет график contributions в течение нескольких минут после push
- Не рекомендуется использовать на публичных репозиториях с реальными проектами

## Примеры использования

### Заполнить график за последний год

Измените в `config.json`:
```json
{
  "fill_contributions": {
    "start_date": "2023-01-01",
    "end_date": "2023-12-31"
  }
}
```

### Заполнить только будние дни

```json
{
  "fill_contributions": {
    "weekend_activity": false
  }
}
```

## Устранение неполадок

**Ошибка: "Путь к репозиторию не существует"**
- Убедитесь, что путь в `config.json` указан правильно
- Или запустите программу из директории репозитория

**Ошибка: "Не удалось отправить изменения на GitHub"**
- Проверьте, что токен указан в `.env` файле
- Убедитесь, что токен имеет права `repo`
- Проверьте подключение к интернету

**Contributions не отображаются на GitHub**
- Убедитесь, что изменения были отправлены (push)
- Подождите несколько минут - GitHub обновляет график не мгновенно
- Проверьте, что коммиты сделаны в правильной ветке
